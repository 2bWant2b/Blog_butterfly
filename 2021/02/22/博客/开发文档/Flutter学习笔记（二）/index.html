# 有状态组件

若想改变页面数据，就用有状态组件。在 IDEA 中键入 stf 快速创建有状态组件

class HomeContent extends StatefulWidget {
  @override
  _HomeContentState createState() => _HomeContentState();
}
class _HomeContentState extends State<HomeContent> {
  int countNum=0;
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text("${this.countNum}"),
        RaisedButton(
            child: Text("加1"),
            textColor: Theme.of(context).accentColor,
            onPressed: (){
              setState(() {
                countNum++;
              });
            }
        ),
        RaisedButton(
            child: Text("减1"),
            textColor: Theme.of(context).accentColor,
            onPressed: (){
              setState(() {
                countNum--;
              });
            }
        ),
      ],
    );
  }
}

这就和创建项目时的 Demo 是差不多的效果

# bottomNavigationBar

底部导航栏组件，是 Scaffold 里的属性，可以实现页面的切换。先看看项目目录，这里把四个页面和导航组件抽离了出来：

![屏幕截图 2021-02-22 143855](https://gitee.com/tzh363231879/picgo/raw/master/ 屏幕截图 2021-02-22 143855.png)

页面组件是很简单的文字，而且是有状态组件

import 'package:flutter/material.dart';

class HomePage extends StatefulWidget {
  @override
  _HomePageState createState() => _HomePageState();
}

class _HomePageState extends State<HomePage> {
  @override
  Widget build(BuildContext context) {
    return Center(
      child: Text("主页界面",style: TextStyle(fontSize: 40),),
    );
  }
}

然后就是重点的 Tabs ，它返回了 Scaffold 组件，看源码：

class Tabs extends StatefulWidget {
  @override
  _TabsState createState() => _TabsState();
}

class _TabsState extends State<Tabs> {
  int _currentIndex=0;
  List _pageList=[
    HomePage(),
    CategoryPage(),
    SettingPage(),
    MenuPage(),
  ];
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Flutter Demo')),
      body: this._pageList[this._currentIndex],
      bottomNavigationBar: BottomNavigationBar(
        items: [
          BottomNavigationBarItem(icon: Icon(Icons.home),label: "主页"),
          BottomNavigationBarItem(icon: Icon(Icons.category),label: "分类"),
          BottomNavigationBarItem(icon: Icon(Icons.settings),label: "设置"),
          BottomNavigationBarItem(icon: Icon(Icons.menu),label: "菜单"),
        ],
        type: BottomNavigationBarType.fixed,  //设置多个图标显示
        currentIndex: this._currentIndex,
        onTap: (int index){
          setState(() {
            this._currentIndex=index;
          });
        },
      ),
    );
  }
}

bottomNavigationBar 需要写 items 数组，如果导航栏超过三个 Item ，就要设定 type 属性。 ** currentIndex** 是一个索引值，从左到右从 0 开始代表哪一个图标，我们通过这个属性就可以实现选中效果和页面跳转效果了。使用的方法就是 onTap 方法，里面传入一个 setState 方法用于动态改变组件中的属性，这里是改变索引值。 currentIndex: this._currentIndex, 这样就可以改变选中效果了。我们定义了一个 _pageList 数组存放页面组件，通过 body: this._pageList[this._currentIndex], 就可以切换页面了。最后在 main.dart 中写入 home: Tabs(), 即可。

# 路由 (Route)

即原生开发中的 Activity ，重点在于管理路由，实现跳转和关闭页面。

# 基本路由

首先先新建一个待跳转的页面 SearchPage

class SearchPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text("搜索页面"),),
      body: Center(
        child: Text("内容区域",style: TextStyle(fontSize: 40),),
      ),
    );
  }
}

这里和上一节略有不同，因为导航栏点击是改变的 body ，而路由是把整个 SearchPage 显示上来，要配置一整套页面设置

然后 Home 页放了一个 Button :

RaisedButton(
            color: Theme.of(context).accentColor,
            child: Text("跳转到搜索页面"),
            onPressed: (){
              Navigator.of(context).push(
                  MaterialPageRoute(
                      builder: (context)=>SearchPage()
                  )
              );
            }
        ),

Navigator.of(context).push 就可以实现一个跳转的方法
跳转时还可以进行传值操作，例如先创建一个 Form 页面：

class FormPage extends StatelessWidget {
  String formContent;
  FormPage({this.formContent="默认内容"});
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      floatingActionButton: FloatingActionButton(
        child: Text("back"),
        onPressed: (){
          Navigator.of(context).pop();
        },
      ),
      appBar: AppBar(title: Text("表单页面"),),
      body: ListView(
        children: [
          ListTile(leading: Icon(Icons.access_time),title: Text(this.formContent),),
          ListTile(leading: Icon(Icons.access_time),title: Text(this.formContent),),
          ListTile(leading: Icon(Icons.access_time),title: Text(this.formContent),),
          ListTile(leading: Icon(Icons.access_time),title: Text(this.formContent),),
        ],
      ),
    );
  }
}

这里列表里的 title 是可以传入的，在 Home 页写一个 Button :

RaisedButton(
            child: Text("跳转到表单页面并且传值"),
            color: Theme.of(context).accentColor,
            onPressed: (){
              Navigator.of(context).push(
                MaterialPageRoute(builder: (context)=>FormPage(formContent: "传入数据",))
              );
            }
        ),

floatingActionButton 是实现了右下角的圆形按钮， Navigator.of(context).pop(); 实现了退出当前页面。

# 命名路由

命名路由用于统一管理路由，它定义在 **MaterialApp ()** 方法里面

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Tabs(),
      routes: {
        'search':(context)=>SearchPage(),
        'form':(context)=>FormPage(),
      },
    );
  }
}

在这里配置好路由名字以及跳转的组件，同时要在 main.dart 里面引入这两个包
跳转方法也非常简单，如下：

RaisedButton(
            color: Theme.of(context).accentColor,
            child: Text("跳转到搜索页面"),
            onPressed: (){
              Navigator.pushNamed(context, 'search');
            }
        ),

在这里就不必引入包了，因为在全局里已经引入。

关于命名路由的传值，可以查阅中文版官方文档，这里是最新的方法

class SearchPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    //获取路由参数
    var args=ModalRoute.of(context).settings.arguments;
    
    return Scaffold(
      appBar: AppBar(title: Text("搜索页面"),),
      body: Center(
        child: Text("内容区域$args",style: TextStyle(fontSize: 40),),
      ),
    );
  }
}

这里直接在路由页写上 var args=ModalRoute.of(context).settings.arguments; ，其中 args 就是获取的传参。然后看传参代码：

RaisedButton(
            color: Theme.of(context).accentColor,
            child: Text("跳转到搜索页面"),
            onPressed: (){
              //Navigator.pushNamed(context, 'search');
              Navigator.of(context).pushNamed("search", arguments: "hi");
            }
        ),

这里代码仅仅变化了一点

# 替换路由

比如我们正处于 First 路由页，然后通过 Navigator.of(context).pushReplacementNamed('Second') 跳转到了 Second 路由页，这时前一个路由页已经被替换了，点击 Second 页的返回按钮就可以返回到 First 的前级路由了。
如果想返回根路由还可以通过以下代码：

Navigator.of(context).pushAndRemoveUntil(
	new MaterialPageRoute(builder:(context)=> new Tabs()),		
);

这样就可以返回到指定根路由了

# AppBar

自定义 AppBar ，可添加 icon，还可添加顶部导航栏。在 Scaffold 外层添加 DefaultTabController 即可使用 ** bottom: TabBar** 来定义顶部导航栏，一定要设定好 length 个数与后面相对应。在 Appbar 内设置 **leading ** 和 actions 属性中的 IconButton 可实现顶部图标。 bottom: TabBar 中设定的 tabs 列表顺序对应 body 中 TabBarView 的子元素顺序。

class AppBarDemoPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return DefaultTabController(
        length: 3,
        child: Scaffold(
          appBar: AppBar(
            centerTitle: true,
            title: Text("AppBarDemoPage"),
            backgroundColor: Colors.redAccent,
            leading: IconButton(
                icon: Icon(Icons.add),
                onPressed: (){}
            ),
            actions: [
              IconButton(icon: Icon(Icons.label), onPressed: (){})
            ],
            bottom: TabBar(
                indicatorColor: Colors.amber,
                indicatorWeight: 5,
                labelColor: Colors.black,
                unselectedLabelColor: Colors.white,
                labelStyle: TextStyle(fontSize: 15),
                tabs: [
                  Tab(text: "资讯"),
                  Tab(text: "通知"),
                  Tab(text: "时事",)
                ]
            ),
          ),
          body: TabBarView(
              children: [
                Center(child: Text("firstTab",style: TextStyle(fontSize: 20),),),
                Center(child: Text("secondTab",style: TextStyle(fontSize: 20),),),
                Center(child: Text("thirdTab",style: TextStyle(fontSize: 20),),),
              ]
          ),
        )
    );
  }
}

# Drawer 侧边栏

Drawer 写在 Scaffold 组件里面，这里写了一个 Column，其中第一项 Header 使用了 UserAccountsDrawerHeader 来实现类似用户信息的样式，后面紧接了三个 ListTile， Divider(), 是设置分割线。在 ListTile 中还可以写 onTap: (){} 来实现路由跳转， Navigator.popAndPushNamed(context, 'appBarDemo'); 的意思是跳转路由后并使当前的侧边栏退出

return Scaffold(
  drawer: Drawer(
    child: Column(

        Row(
