# Introduction

在 UDN 中，用户被许多 BS 覆盖，但是用户却不知道它们的负载量，所以如何选择最佳的计算卸载方案是本文的重点。

# 系统模型

# 网络模型

  1. 用户层：用户随机分发，每个用户有一个需要计算的迟延敏感任务。一个用户可以与多个 BS 连接，并且通过无线链路向最近的 BS 发送设备信息。

  2. 网络层：每个 UDN 中的分区 cell 包含多个 BS，并且每个 BS 都配备 MEC 移动边缘计算服务。并非所有 BS 之间通过光纤连接，为了研究都设定为无线连接。

  3. 控制层：包括一个宏 BS。宏基站不仅配备了一个 MEC 设备，还有一个 SDN 控制器。网络层的 BS 通过回程链路连接到宏基站。SEC-UDN 控制器支持网络层和用户层的覆盖范围。对于超负荷的 BS，控制器可以感知任务情况，做出最佳决策然后返回给用户。控制器周期性地更新信息表，它包括任务信息表（task）：任务类型，任务数据量和任务计算金额；移动设备信息表（mobile device）：终端设备的剩余功率，CPU 周期和其他相关数据；BS 信息表：BS 无线电接入负载和 BS 负载。用户会定期将自己的设备信息和任务信息发送到最近的 BS，然后 BS 从多个用户收集信息定期更新到控制器上的信息表。
